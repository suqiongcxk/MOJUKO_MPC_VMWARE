1  系统架构
================================================================================

  3 个 ROS2 节点:

    user_command ──(步态+目标)──> legged_robot_sqp_mpc ──(关节力矩)──> mujoco_simulator
      (键盘输入)    <──(观测)──    (MPC+WBC+卡尔曼)    <──(状态)──      (物理引擎)

================================================================================
2  MPC 黑盒接口 (24维状态 + 24维控制)
================================================================================

  输入 (3个):
    [1] 机器人状态    SimulatorStateData     基座pose/vel + 12关节pos/vel + 接触标志
    [2] 步态模式      GaitModeSchedule       时间序列: [0.0s LF_RH, 0.3s RF_LH, ...]
    [3] 目标轨迹      MpcTargetTrajectories  键盘 "goal:x y z yaw" 转换的两帧目标

  输出 (2个):
    [1] 关节控制指令  JointControlData       [12关节力矩, 12关节位置, 12关节速度]
    [2] MPC观测      MpcObservation          当前 [v_com, L/mass, pose, joints]

  状态 x (24维):                  控制 u (24维):
    [0:2]   v_com (质心线速度)       [0:11]  4足接触力 (3D×4)
    [3:5]   L/mass (归一化角动量)   [12:23] 4足端速度 (3D×4, 相对基座)
    [6:8]   基座位置
    [9:11]  基座姿态 ZYX欧拉
    [12:23] 12关节角度

================================================================================
3  代价函数与约束
================================================================================

  min  J = Σ (x-x_ref)ᵀQ(x-x_ref) + uᵀRu

  Q 权重: 位置(1000~1500) > 姿态(100~500) > 关节(10~20)
  R 权重: 足端力(小,允许用力) << 足端速度(大,限制乱晃)

  约束: 摩擦锥 |F_xy|≤μF_z (μ=0.3), 自碰撞(8对link, 间距≥0.05m), 力矩≤330.5Nm

================================================================================
4  质心动量矩阵 Ag — 核心转换
================================================================================

  作用: h = Ag(q) · v    (6D 质心动量 ← 18D 广义速度)

  结构: Ag = [Ab(6×6) | Aj(6×12)]
    Ab: 基座速度 → 动量  (含质量+惯量信息)
    Aj: 关节速度 → 动量  (含腿部质量分布信息)

  关键性质:
    - Ag = Ag(q) 不是常数, 随构型变化 (腿摆动→质量分布变→Ag变)
    - Ag = 运动学(雅可比列) × 动力学(CRBA累积惯量), 是"带权重的雅可比"
    - 18维→6维的降维保留了控制所需的全部整体运动信息

  为什么用 Ag 降维?
    1. MPC 只需规划整体运动, 不需要管每个关节怎么动 (留给WBC)
    2. 质心动量动力学线性: ḣ = Σf + mg (18维关节动力学是非线性的)
    3. 步态无关: ḣ=Σf+mg 不管几条腿着地都成立
    4. MPC(慢,67Hz,规划) ↔ WBC(快,400Hz,执行) 的自然分层接口

================================================================================
5  状态估计: 质心动量是怎么得到的?
================================================================================

  仿真 (直接读真相):
    MuJoCo framelinvel/angvel 真值 → computeCentroidalStateFromRbdModel()
    → h = Ag·v / mass → [v_com, L/mass]

  真机 (传感器融合):
    IMU + 关节编码器 + 接触 → 卡尔曼滤波(18维,估计base_pos/vel+足端位置)
    → computeCentroidalStateFromRbdModel()  (与仿真共用同一函数!)
    → h = Ag·v / mass → [v_com, L/mass]

  卡尔曼滤波技巧: 支撑足位置不变(虚拟速度=0观测) → 校正base_vel估计
                 非着地腿观测噪声×100 → 摆动腿不参与校正

================================================================================
6  完整控制链路 (一个周期)
================================================================================

  ┌──────────────┬───────────────────────────────────────────────────┐
  │ 步骤           │ 内容                                             │
  ├──────────────┼───────────────────────────────────────────────────┤
  │ 0 传感器       │ IMU + 关节编码器 + 接触 → 原始数据                 │
  │ 1 状态估计     │ 卡尔曼滤波 → estimatedRbdState (base_vel等)       │
  │ 2 Ag转换       │ Ag·v/mass → [v_com, L/mass] → MPC 24维状态       │
  │ 3 MPC预测优化  │ h_next = h + dt·(Σf+mg), 选最优力序列 (67步预测)  │
  │ 4 Ag逆运算     │ v_base = Ab⁻¹(h_opt - Aj·q̇_joint)               │
  │ 5 WBC力矩分配  │ QP: min Σw||task||², 满足约束 → 12关节力矩        │
  │ 6 执行          │ 力矩→电机→物理运动→回到0                          │
  └──────────────┴───────────────────────────────────────────────────┘

  核心动力学: d(v_com)/dt = (Σf_i)/m + g
  即: MPC 通过选择最优的足端力来控制质心加速度 → 速度 → 位置
